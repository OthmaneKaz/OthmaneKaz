You finish your 20-hour long JavaScript tutorial “Zero to Hero” and you go open up VSCode and BAM! you can't do anything, you feel stuck. This cycle is known as "tutorial hell,"and it's a common experience among programmers.

Prerequisites

  • Choose something you want to learn: It can be a general-purpose programming language like C or Python, or a tech stack like a frontend stack (HTML, CSS & JavaScript) or a backend stack (Node.js (JavaScript) with Express.js or Django (Python)).
  • Basic Language Syntax: Ensure you understand variables, loops, conditionals, functions, etc.
  • Commit: Promise yourself that you will not watch another tutorial until you've completed the exercises. No Coursera, no Udemy, no tutorials or courses or books until you're done (or until never).

Part I: Warming up the Gears

Once you get all the prerequisites settled (which should not take you more than a couple of hours, maybe an hour or two), you should understand what Question-Driven Development is. Question-Driven Development, or QDD in short, is a framework inspired by programmer Nick Jantakis. It mirrors real-life software development by encouraging you to set a goal and break it down into small, manageable questions that you can answer by researching and problem-solving.

Here's how to apply QDD:

Step 1: Start with Small Projects

Aim for projects with less than 100 lines of unique code (excluding boilerplate). Start with simple problems, such as:
  • Simple Calculator in C.
  • To-Do List App.
  • Simple URL Shortener.
These problems are small but effective in warming up your brain without overwhelming you. Over time, they will build up into full-blown projects.

Step 2: Use Google as a Tool

For each problem, break it down into smaller questions. For example, to solve the palindrome problem, you might start with:
  • How do I set up a C programming environment?
  • How do I read user input in C?
  • How do I perform basic arithmetic operations in C?
As you search for solutions, you may copy code snippets, but the key is to integrate them into your code and make them work for your specific problem. This iterative process of trial and error is essential for learning.

Step 3: Complete 50 Small Projects

Aim to complete a minimum of 50 small projects using QDD. This practice helps reinforce your coding skills and establishes a positive feedback loop, making coding fun and engaging.

Part II: Move to Intermediate Projects

Once you're comfortable with small projects, challenge yourself by cloning a tool, an app, or a website that you find interesting. Build a compiler or a text editor (mainly you will use C or C++ if you want to up the challenge) or choose 10 websites and try to replicate their design and functionality. Cloning has several benefits:

  • No Procrastination: You won't get stuck in decision-making because the design and/or functionality are already defined.
  • Skill Building: Cloning helps you understand how different components work together, leading to the development of original projects later in your career.

Defining Your Minimum Viable Product (MVP)

For each cloning project, it's essential to define your MVP to know when your project is complete. An MVP outlines the basic features necessary to consider the project functional and complete. Here's how you can approach this for different types of projects:

Example 1: Cloning the Reddit Landing Page
MVP Features:
  • Navbar: Includes a search bar, categories, and a logo.
  • Sidebar: Contains basic links to various sections.
  • Main Feed Area: Displays a list of posts or content.
Breakdown into Smaller Parts: Search Bar:
  • How do I create a search bar in HTML and CSS?
  • How do I add an icon next to the search bar?
Navbar:
  • How do I position elements within a navbar using CSS?
  • How do I add responsive design features to the navbar?
Sidebar:
  • How do I create a vertical navigation menu in CSS?
  • How do I style links within the sidebar?
Example 2: Building a Simple Compiler in C
MVP Features:
  • Source Code Input: Accepts input in a simple programming language or subset of C.
  • Lexical Analysis: Tokenizes the input into meaningful symbols.
  • Syntax Analysis: Checks the grammar and structure of the code.
  • Code Generation: Outputs simple machine code or intermediate representation.
Breakdown into Smaller Parts: Source Code Input:
  • How do I read input from a file in C?
  • How do I handle different input formats?
Lexical Analysis:
  • How do I tokenize a string in C?
  • How do I handle whitespace and special characters in tokenization?
Syntax Analysis:
  • How do I implement a simple parser in C?
  • How do I define and check grammar rules?
Code Generation:
  • How do I translate tokens into machine code?
  • How do I handle errors during code generation?

The Importance of Decomposition

This process of decomposition is vital in software development, as it enables you to tackle large problems by solving smaller, manageable tasks. Breaking down projects into smaller parts keeps you motivated by providing a clear list of achievable goals. It allows you to focus on one component at a time, making the overall task less daunting and more structured.

By defining clear MVPs and breaking down each feature into actionable questions, you can approach complex projects methodically and build your skills step by step. This method not only helps you learn effectively but also builds confidence as you see tangible progress in your projects.

Final Takes

After completing these exercises, you’ll have a better understanding of how to approach coding problems independently. While tutorials are always a resource, it's important to build a healthy relationship with them.

A general rule here is that after completing a tutorial section, create three small projects using the concepts you've learned. For example, after a tutorial on forms, create three simple forms to reinforce your understanding.

Remember! To learn something big, break it down and learn each small part of it and apply what you learn, eventually you will master it (applicable to any skill you want to learn).
